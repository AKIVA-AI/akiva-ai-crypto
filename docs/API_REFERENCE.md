# Enterprise Crypto — API Reference

**Version:** 1.0
**Date:** 2026-03-19
**Base URL:** `http://localhost:8000`

---

## Authentication

All API endpoints (except health, metrics, docs, and `/auth`) require a Bearer JWT token in the `Authorization` header.

```text
Authorization: Bearer <supabase-jwt-token>
```

The auth middleware extracts the token, verifies it against Supabase, and attaches `user_id` and `user_role` to the request state. Roles are: `admin`, `cio`, `pm`, `trader`, `analyst`, `viewer`.

**Unauthenticated requests** receive `401 Authentication failed`.

**Insufficient permissions** receive `403 Insufficient permissions` (for role-restricted endpoints).

---

## Rate Limiting

Rate limits are enforced per IP address via slowapi. Exceeding a limit returns `429 Too Many Requests`.

| Category | Limit | Applied to |
| -------- | ----- | ---------- |
| Trading | 30/minute | Order placement, kill switch, trade operations |
| Read | 100/minute | GET endpoints (positions, orders, market data) |
| Write | 60/minute | General POST/PUT operations |
| Auth | 10/minute | Authentication endpoints |
| WebSocket | 5/minute | WebSocket connection upgrades |
| Default | 200/minute, 1000/hour | All other endpoints |

## WebSocket (`/ws`)

### WS /ws/stream/{stream_type}

Real-time data streaming via WebSocket.

**Authentication:** Required via `?token=<jwt>` query parameter or `Authorization: Bearer <jwt>` header. Unauthenticated connections are closed with code 4001.

**Stream types:**

| Type | Data |
| ---- | ---- |
| market | Real-time price updates |
| signals | Trading signals from strategies |
| arbitrage | Arbitrage opportunities |
| portfolio | Portfolio updates (user-specific) |
| agents | Agent status updates |
| all | All streams combined |

**Connection flow:**

1. Client connects with auth token
2. Server sends `{ "type": "connected", "stream": "market" }`
3. Server sends data messages: `{ "stream": "market", "timestamp": "...", "data": {...} }`
4. Server sends `{ "type": "ping" }` every 30s if no client message
5. Client can send `{ "type": "subscribe", "symbols": ["BTC/USDT"] }` to filter
6. Client can send `{ "type": "pong" }` in response to ping

## Error Responses

All errors follow a consistent format:

```json
{
  "error": "Internal server error",
  "message": "An unexpected error occurred",
  "timestamp": "2026-03-19T12:00:00+00:00"
}
```

| Status | Meaning |
| ------ | ------- |
| 400 | Bad request (validation error, invalid parameters) |
| 401 | Missing or invalid authentication token |
| 403 | Insufficient role permissions |
| 404 | Resource not found |
| 429 | Rate limit exceeded |
| 500 | Internal server error |
